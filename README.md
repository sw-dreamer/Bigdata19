### 1. 배열 실습 문제

#### 프로젝트 개요
이 프로젝트는 Java 배열을 사용하여 간단한 데이터 처리 및 분석을 수행하는 실습이었습니다. 주어진 문제에서 배열을 활용해 기본적인 알고리즘을 구현했습니다.

#### 사용된 프로그램
- **Eclipse Version: 2024-03 (4.31.0)**

#### 사용된 기술
- **Java**
- **배열**

#### 구현 과정
1. 배열을 생성하여 다양한 데이터를 저장하고, 필요한 작업을 수행.
2. 배열을 순차적으로 탐색하면서 최소값, 최대값 등을 구하는 알고리즘을 작성.
3. 배열의 원소들을 정렬하거나 필터링하는 기능을 구현.

#### 주요 기능
- 배열에서 숫자들의 합, 평균, 최소값 및 최대값 구하기.
- 배열을 오름차순, 내림차순으로 정렬.
- 배열에서 특정 값을 찾는 알고리즘 구현.

#### 결과
- 배열을 활용하여 기본적인 데이터 분석 작업을 수행.
- 알고리즘 구현에 대한 이해도가 향상됨.

### 2. 컬렉션 실습 문제

#### 프로젝트 개요
이 프로젝트는 Java의 **컬렉션 프레임워크**를 활용하여 다양한 데이터 구조를 사용하고, 이를 통해 문제를 해결하는 방법을 학습하는 데 중점을 두었습니다.

**Arraylist**, **HashMap** 등의 컬렉션을 사용하여 데이터를 저장하고 처리하며, 각 컬렉션의 특성과 효율적인 활용 방법을 실습했습니다.

#### 사용된 프로그램
- **Eclipse Version: 2024-03 (4.31.0)**

#### 사용된 기술
- **Java**
- **ArrayList, HashMap (컬렉션 프레임워크)**

#### 구현 과정
1. **ArrayList**:
   - `ArrayList`는 인덱스를 통해 데이터에 빠르게 접근할 수 있기 때문에 순차적인 데이터 관리에 유용합니다.
   - 프로젝트에서 **ArrayList**를 사용하여 데이터를 순차적으로 저장하고 인덱스를 이용해 접근하는 방법을 실습했습니다.
   - 다양한 데이터를 추가, 삭제 및 조회하는 기능을 구현했습니다.

2. **HashMap**:
   - `HashMap`은 키-값 쌍을 저장하여 데이터를 효율적으로 검색할 수 있는 자료 구조입니다.
   - 연락처 관리 프로그램에서 **HashMap**을 사용하여 이름을 키로, 전화번호를 값으로 저장하고, 효율적으로 연락처를 관리하는 방법을 구현했습니다.

#### 주요 기능
- **ArrayList**를 활용한 데이터 저장 및 조회:
   - 사용자 입력에 따라 데이터를 `ArrayList`에 추가하고, 특정 인덱스를 통해 조회하거나 삭제하는 기능을 구현.
  
- **HashMap**을 이용한 연락처 관리:
   - 이름을 키로 하고 전화번호를 값으로 저장하여 연락처를 관리.
   - **연락처 추가, 수정, 삭제, 조회** 기능을 통해 `HashMap`을 실제 데이터 관리에 어떻게 적용할 수 있는지 실습.

#### 결과
- Java 컬렉션 프레임워크의 다양한 자료 구조를 활용하여 문제를 해결하는 방법에 대해 깊이 이해하게 되었으며, 실용적인 데이터 관리 및 처리 방법을 익혔습니다.
- **Arraylist**, **HashMap** 각각의 특성과 그에 적합한 사용 사례를 파악하고, 실제 문제에 어떻게 적용할 수 있는지를 실습했습니다.
  
#### 학습 성과
- **ArrayList**를 사용한 순차적 데이터 관리와 검색에 대한 이해.
- **HashMap**을 활용하여 효율적인 키-값 쌍 데이터 관리 및 검색 기능을 구현할 수 있었습니다. 

이를 통해 Java의 컬렉션 프레임워크에 대한 실용적인 이해도를 높였으며, 다양한 실세계 문제를 해결할 수 있는 기술적 기초를 다졌습니다.

### 3. HashMap을 이용한 연락처 관리 프로그램

#### 프로젝트 개요
이 프로젝트는 **HashMap**을 사용하여 연락처 정보를 관리하는 프로그램을 구현하는 것이 목표였습니다. 사용자로부터 입력을 받아 연락처를 추가, 수정, 삭제하고 조회하는 기능을 제공하는 프로그램을 만들었습니다.

#### 사용된 프로그램
- **Eclipse Version: 2024-03 (4.31.0)**
  
#### 사용된 기술
- **Java**
- **HashMap**
- **Scanner (입력 처리)**

#### 구현 과정
1. **HashMap을 사용한 연락처 저장**: 이름을 키로 하고, 전화번호를 값으로 저장하는 방식으로 연락처를 관리.
2. **기능 구현**:
    - **연락처 추가**: 사용자로부터 이름과 전화번호를 입력받아 저장.
    - **연락처 수정**: 이름을 통해 기존 연락처를 수정.
    - **연락처 삭제**: 이름을 통해 특정 연락처 삭제.
    - **연락처 조회**: 이름을 통해 특정 연락처 조회 및 출력.
3. **메뉴 제공**: 사용자에게 다양한 기능을 선택할 수 있는 메뉴를 제공하고, 선택에 따라 작업을 수행.

#### 주요 기능
- 연락처 추가, 수정, 삭제, 조회 기능.
- 잘못된 입력에 대한 예외 처리.
- 사용자 입력에 따라 프로그램 종료 또는 반복 실행.

#### 결과
- HashMap을 활용하여 데이터를 효율적으로 관리하고, 다양한 기능을 구현하여 실용적인 연락처 관리 프로그램을 완성.
- 실시간 데이터 관리에서 HashMap의 효율성과 중요성을 이해.

### 4. 데이터베이스 1차 실습문제

#### 프로젝트 개요

이 프로젝트는 영화 대여 시스템의 다양한 데이터를 분석하는 문제들을 해결하는 것을 목표로 합니다. SQL을 활용하여 매장별 매출 분석, 특정 배우의 매출 기여도 분석, 대여 및 반환 서비스 속도 분석 등 여러 가지 분석을 수행합니다. 이를 통해 경영 및 운영 전략 수립에 필요한 데이터를 도출합니다.

#### 사용된 프로그램
- **MySQL 8.0.17**
- **DBeaver**
  
#### 사용된 기술
- **SQL**: 데이터베이스 쿼리 작성 및 분석
- **집계 함수**: 데이터 집계 및 계산
- **JOIN**: 여러 테이블 간 데이터 결합
- **WHERE**: 조건을 설정하여 데이터 필터링

#### 주요 기능
1. **매장별 월별 매출 분석**: 매장별 월별 매출을 집계하여 매출 추이를 분석합니다.
2. **특정 배우의 매출 기여도 분석**: 특정 배우가 출연한 영화들의 매출 기여도를 계산합니다.
3. **대여 및 반환 서비스 속도 분석**: 매장별 대여 후 반환까지 걸린 평균 시간을 계산하여 서비스 속도를 분석합니다.
4. **대여되지 않은 영화 목록 찾기**: 대여되지 않은 영화를 찾아 불필요한 재고를 줄이는 전략을 세울 수 있습니다.
5. **고객의 활동성 분석**: 고객별 대여 횟수와 결제 금액을 분석하여 VIP 고객을 식별합니다.
6. **특정 카테고리의 대여 트렌드 분석**: 특정 장르의 대여 트렌드를 분석하여 마케팅 전략을 수립합니다.
7. **특정 기간 동안 대여된 영화 목록**: 특정 기간 동안 대여된 영화를 분석하여 인기 영화를 확인합니다.
8. **고객의 평균 대여 간격 분석**: 고객별 평균 대여 간격을 계산하여 맞춤형 서비스를 제공합니다.

#### 결과
- **매장별 매출 분석**: 각 매장의 월별 매출을 분석하여 매출 추이를 파악하고, 마케팅 전략을 세울 수 있습니다.
- **특정 배우의 매출 기여도 분석**: 배우별 영화가 전체 매출에서 차지하는 비율을 파악하여 향후 전략을 세울 수 있습니다.
- **대여 및 반환 서비스 속도 분석**: 서비스 속도가 빠른 매장을 파악하여 고객 만족도를 높이고, 효율적인 운영을 할 수 있습니다.
- **대여되지 않은 영화 목록**: 대여되지 않은 영화를 찾아 불필요한 재고를 줄이고 프로모션을 최적화할 수 있습니다.
- **VIP 고객 분석**: 대여 횟수와 결제 금액을 기반으로 VIP 고객을 식별하여 타겟 마케팅 전략을 수립할 수 있습니다.
- **카테고리별 대여 트렌드 분석**: 장르별 대여 트렌드를 분석하여 특정 장르에 대한 마케팅 전략을 세울 수 있습니다.

### 5. 데이터베이스 2차 실습문제

#### 프로젝트 개요

이 프로젝트는 영화 대여 시스템의 데이터를 SQL의 분석 함수와 순위 함수를 활용하여 다양한 분석을 수행하는 것을 목표로 합니다. 영화의 대여 횟수, 배우별 인기, 고객의 활동성 등을 순위별로 분석하여 경영 전략에 필요한 인사이트를 제공합니다.


#### 사용된 프로그램
- **MySQL 8.0.17**
- **DBeaver**
  
#### 사용된 기술
- **SQL (순위 함수)**: 데이터의 순위를 계산하는 함수 사용
- **JOIN**: 여러 테이블 간 데이터 결합
- **GROUP BY**: 그룹화하여 데이터를 집계
- **ORDER BY**: 결과를 정렬하여 순위 계산

#### 주요 기능
1. **각 배우의 총 출연 횟수와 순위 계산**: 각 배우가 출연한 횟수를 계산하여 순위를 매깁니다.
2. **각 영화의 총 대여 횟수와 순위 계산**: 영화별 대여 횟수를 분석하고 순위를 매깁니다.
3. **각 고객의 대여 횟수와 순위 계산**: 고객별 대여 횟수를 계산하고 순위를 매깁니다.
4. **대여가 가장 많은 직원 순위 계산**: 직원별 대여 건수를 계산하여 순위를 매깁니다.
5. **각 카테고리에서 가장 많이 대여된 영화 순위 계산**: 각 장르별로 가장 많이 대여된 영화를 분석합니다.
6. **각 배우의 출연 영화 평균 대여 횟수 순위 계산**: 배우별로 출연한 영화의 평균 대여 횟수를 계산하여 순위를 매깁니다.
7. **대여 횟수가 가장 많은 고객을 나라별로 구분하여 순위 계산**: 국가별로 대여 횟수가 많은 고객을 분석합니다.
8. **대여 금액이 가장 높은 고객 TOP 10 계산**: 대여 금액이 높은 상위 10명의 고객을 파악합니다.
9. **매월 총 수익 계산과 월별 순위**: 월별 수익을 계산하고 순위를 매깁니다.
10. **대여된 영화의 평균 수익과 순위 계산**: 각 영화의 대여 수익을 분석하여 순위를 매깁니다.

#### 결과
- **각 배우의 출연 횟수와 순위**: 출연 횟수가 많은 배우의 활동 패턴을 분석하여 향후 전략을 수립할 수 있습니다.
- **각 영화의 대여 횟수와 순위**: 대여 횟수가 많은 영화를 파악하여 인기 영화를 중심으로 프로모션을 할 수 있습니다.
- **각 고객의 대여 횟수와 순위**: 대여 횟수가 많은 고객을 식별하여 타겟 마케팅 전략을 세울 수 있습니다.

#### 사용된 프로그램

- **Eclipse Version: 2024-03 (4.31.0)**
  
#### 주요 기능
1. **연락처 추가**  
   사용자로부터 이름, 전화번호, 이메일 등을 입력받아 연락처를 데이터베이스에 저장합니다.

2. **연락처 조회**  
   저장된 모든 연락처 정보를 출력하거나 특정 조건에 맞는 연락처를 검색합니다.

3. **연락처 수정**  
   사용자가 지정한 연락처의 정보를 수정합니다.

4. **연락처 삭제**  
   사용자가 지정한 연락처를 삭제합니다.

#### 기술 스택
- **Java**  
  - `ArrayList`를 이용하여 연락처 데이터를 관리
  - JDBC를 통해 MySQL과 연결
  - **MVC 패턴** 적용
- **MySQL**  
  - 연락처 데이터를 저장하기 위한 데이터베이스
- **JDBC**  
  - Java에서 MySQL 데이터베이스와의 연결을 위한 기술

#### MVC 패턴 구조

1. **Model (모델)**  
   - 모델은 프로그램의 데이터 구조와 비즈니스 로직을 담당합니다.
   - MySQL 데이터베이스와의 상호작용을 관리하며, 연락처의 추가, 수정, 삭제, 조회 등의 기능을 구현합니다.

   - 사용자의 입력을 처리하고, 해당 입력에 맞는 로직을 실행하는 역할을 합니다.

2. **입력 처리 (Controller)**
   - 컨트롤러는 사용자의 명령을 받아 필요한 로직을 실행합니다.
   - 예를 들어, 연락처 추가 요청 시, 해당 데이터를 모델로 전달하여 데이터베이스에 저장하도록 합니다.

3. **데이터 처리 (Model)**
   - 모델은 데이터베이스와의 연결을 통해 데이터를 저장하거나 조회합니다.
   
4. **결과 출력 (View)**
   - 모델에서 처리한 데이터를 컨트롤러를 통해 뷰에 전달하여 사용자에게 결과를 표시합니다.

## 8. MySQL DB를 활용한 웹 기반 연락처 관리 프로그램

이 프로젝트는 MySQL 데이터베이스를 사용하여 웹 기반 연락처 관리 시스템을 구현한 것입니다. 사용자는 웹 페이지를 통해 연락처 정보를 추가, 수정, 삭제 및 조회할 수 있으며, 회원가입과 로그인 기능을 통해 인증을 수행하고 개인별 연락처를 관리할 수 있습니다. Java와 Spring Framework를 활용해 데이터는 MySQL에 영구적으로 저장되며, 사용자 인터페이스는 Spring Boot를 이용해 구현되었습니다.

#### 사용된 프로그램
- **MySQL 8.0.17**
- **DBeaver**
- **IntelliJ IDEA 2024.3.2.2 (Community Edition)**

### 주요 기능

#### 1. **회원가입**
- 사용자는 웹 폼을 통해 이름, 이메일, 비밀번호 등을 입력하여 회원가입을 진행합니다.
- 회원가입 정보는 MySQL 데이터베이스에 저장됩니다.

#### 2. **로그인**
- 사용자는 이메일과 비밀번호를 통해 로그인하고, 로그인 후 자신의 연락처 정보를 조회하고 관리할 수 있습니다.

#### 3. **연락처 추가**
- 사용자는 웹 폼에서 이름, 전화번호, 이메일 등의 정보를 입력하여 연락처를 추가합니다.
- 입력된 데이터는 MySQL 데이터베이스에 저장됩니다.

#### 4. **연락처 조회**
- 저장된 모든 연락처 정보를 웹 페이지에서 확인하거나, 특정 조건에 맞는 연락처를 검색할 수 있습니다.

#### 5. **연락처 수정**
- 사용자는 연락처 정보를 수정할 수 있으며, 수정된 내용은 MySQL 데이터베이스에 반영됩니다.

#### 6. **연락처 삭제**
- 사용자는 연락처 목록에서 선택한 항목을 삭제할 수 있습니다.

### 기술 스택

- **Spring Boot**
  - **Spring Web 사용** : HTTP 요청과 응답을 처리할 수 있는 기능을 제공하여 웹 애플리케이션을 구축할 때 클라이언트의 요청을 받아 처리하고, 그에 대한 응답을 반환하는 데 사용
  - **Mustache 템플릿 엔진 사용** : Mustache 템플릿 엔진을 사용하여 사용자 인터페이스(UI)를 구성하여  웹 페이지에서 회원가입/로그인 또는 연락처 목록 페이지를 구현
  - **Spring Data JPA 사용** : MySQL 데이터베이스와 연동하여 연락처 및 사용자 데이터를 관리
    
- **MySQL**
  - 연락처 데이터를 저장하는 데이터베이스
  - 사용자 정보, 로그인, 연락처 데이터를 저장하는 테이블

### 패턴 구조

#### 1. **Model (모델)**
- 프로그램의 데이터 구조와 비즈니스 로직을 담당하며, 데이터베이스와의 상호작용을 관리합니다.
- MySQL과 연동하여 연락처의 추가, 수정, 삭제, 조회, 회원가입, 로그인 등의 기능을 구현합니다.

#### 2. **View (뷰)**
- 사용자와 상호작용하는 웹 페이지를 처리합니다.
- HTML과 JSP를 사용하여 사용자에게 입력 양식과 결과를 표시합니다.

#### 3. **Controller (컨트롤러)**
- 사용자의 입력을 처리하고, 해당 입력에 맞는 로직을 실행합니다.
- 모델과 뷰 사이의 상호작용을 조정하며, 사용자가 요청한 데이터를 뷰에 전달하거나 모델에 데이터를 전달합니다.

#### 프로그램 흐름

1. **사용자 입력**
   - 웹 페이지에서 사용자로부터 연락처 추가, 수정, 삭제, 조회, 회원가입, 로그인 등의 명령을 입력받습니다.
   - 예를 들어, 사용자는 웹 폼에서 이름, 전화번호, 이메일을 입력하여 새로운 연락처를 추가할 수 있습니다.

2. **입력 처리 (Controller)**
   - 컨트롤러는 사용자의 요청을 받아 필요한 로직을 실행합니다.
   - 예를 들어, 회원가입 요청 시, 입력된 정보를 모델로 전달하여 사용자 정보를 MySQL에 저장하고, 로그인 요청 시 입력된 정보와 데이터베이스의 정보를 비교하여 인증을 진행합니다.
   - 연락처 추가, 수정 및 삭제 요청도 해당 로직을 처리합니다.
